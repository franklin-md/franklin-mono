**3. Needs New Franklin API or Protocol Work**
- Per-session tool/extension allowlisting is the biggest API gap. `pi-subagents` agent configs include `tools`, `extensions`, and direct MCP-tool selections ([agents.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/.context/external/pi-subagents/agents.ts)); Franklin currently builds one static extension set per `SessionManager`, and tool injection serializes all registered tools into every session ([app/index.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/agent/src/app/index.ts), [manager.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/extensions/src/systems/sessions/runtime/manager.ts), [tool-injector.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/extensions/src/systems/core/compile/builders/tool-injector.ts)). Near-full parity wants a first-class per-session tool/profile filter.
- Usage/cost/model-attempt telemetry needs protocol work. `pi-subagents` tracks usage, tokens, and model attempts ([types.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/.context/external/pi-subagents/types.ts), [execution.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/.context/external/pi-subagents/execution.ts)); Franklin’s public Mini-ACP `Message` / `TurnEnd` types do not carry usage, and the Mini-ACP README explicitly calls usage in `TurnEnd` a known gap ([message.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/types/message.ts), [stream.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/types/stream.ts), [README.md](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/README.md)).

**4. Phased Path**
- Phase 1, minimally useful transpilation: transpile agent markdown into Franklin session profiles; support single-run plus sequential chain; use one shared Franklin built-in bundle (`filesystem`, `bash`, `web`, `todo`, `status`, `instructions`); map `fresh`/`fork`, model override, cwd/network override, output/read file conventions.
- Phase 2, still shim-only: add parallel orchestration, async/background jobs, persisted result/status views, agent CRUD, and worktree provisioning in the app layer; add a custom instruction spec for AGENTS-style inheritance.
- Phase 3, near-full parity: add Franklin API support for per-session tool/extension filtering and direct MCP-tool scoping, then extend Mini-ACP with usage/cost/model-attempt telemetry. Until those land, a transpiled version can be useful, but it will have a shared tool surface and weaker observability than `pi-subagents`.
