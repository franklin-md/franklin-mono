2. Runtime/protocol constraints that matter for compatibility

- Mini-ACP’s wire surface is small: `initialize`, `setContext`, streaming `prompt`, notification `cancel`, and reverse-RPC `toolExecute` ([protocol/types.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/protocol/types.ts:5), [manifest.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/protocol/manifest.ts:5)).
- Context starts empty; `CtxPatch` merges `history` and `config` by property but replaces `tools` wholesale ([mini-acp README](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/README.md:82), [ctx-tracker.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/protocol/ctx-tracker.ts:8), [types/context.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/types/context.ts:45)).
- Single active turn is part of the protocol; `setContext` during a turn is unspecified. The adapter currently has TODOs rather than enforcement, and `cancel()` with no current turn is a no-op ([mini-acp README](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/README.md:66), [adapter.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/protocol/adapter.ts:35)).
- Stream invariants matter: `TurnStart`, `Chunk`, `Update`, `TurnEnd`; `Update` is authoritative, and tool calls/results are not stream events ([types/stream.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/src/types/stream.ts:9), [mini-acp README](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/mini-acp/README.md:185)).
- Fork semantics are explicit: core `child()` clears messages but keeps model config; store `child()` uses `fresh`, `fork()` uses `copy`; store sharing can be `shared` or `private` ([core/runtime.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/extensions/src/systems/core/runtime.ts:63), [store/runtime.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/extensions/src/systems/store/runtime.ts:39), [sharing.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/extensions/src/systems/store/api/sharing.ts:1)).
- Agent auth wiring is automatic: `withAuth` resolves provider API keys on `setLLMConfig`, reconnects at build, and live-syncs credential changes ([with-auth.ts](/Users/afv/conductor/workspaces/franklin/yangon-v2/packages/agent/src/auth/with-auth.ts:36)).
